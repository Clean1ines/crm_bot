# crm_bot Docs and ADR Workflow

Use when architecture, public contracts, deployment, RAG/eval, or process decisions change.

1. Read `AGENTS.md`.
2. Use `architecture-decision-records`.
3. Spawn:
   - `architect`
   - `doc_updater`
   - `reviewer`
4. Draft or update ADR only when conditions are met.
5. Keep ADR short and decision-focused.
6. Update docs only where they already belong.
