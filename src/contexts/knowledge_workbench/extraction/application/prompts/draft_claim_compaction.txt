NODE: draft_claim_compaction

INPUT := one JSON object.
TASK := compact raw draft claims into fewer, richer, non-duplicating claims.
OUTPUT := exactly one JSON object.
NEVER output text outside JSON.

INPUT_SCHEMA:
{
  "claims": [
    {
      "id": "",
      "claim": "",
      "questions": []
    }
  ]
}

OUTPUT_SCHEMA:
{
  "compacted_claims": [
    {
      "key": "",
      "claim": "",
      "claim_kind": "definition",
      "source_claim_refs": [],
      "triples": [],
      "merge_decision": "merged"
    }
  ]
}

LANG:

* write `claim` in same language as input claims
* keep enum values in English

claim_kind ∈ {
definition, property, capability, limitation, rule, condition, process, list,
comparison, criterion, example_set, value, exception, other
}

merge_decision ∈ {merged, unmerged}

TRIPLE:
{
  "subject": "",
  "predicate": "",
  "object": "",
  "qualifiers": []
}

predicate ∈ {
is_a, has_capability, has_property, has_value, has_limitation,
requires, supports, integrates_with, used_for, handles, transfers_to,
provides, improves, checks, removes, creates, connects_to
}

COMPACTION_RULES:
IF claims express same core meaning -> merge.
IF claims answer same/nearly same questions with same meaning -> merge.
IF A is fuller version of B -> keep fuller meaning.
IF A overlaps B and adds important content -> one richer claim preserving added content.
IF claims describe different facts/values/limitations/rules/processes -> keep separate.
IF composite claim is one practical answer bundle -> keep composite.
NEVER create vague mega-claim hiding distinct facts.

FIELD_OWNERSHIP_RULES:
NEVER output questions.
NEVER output exclusion_scope.
NEVER output granularity.
exclusion_scope, evidence, possible questions, and granularity are reconstructed later by code from source claims.

TRACE_RULES:
Every input claim.id MUST appear exactly once across all source_claim_refs.
source_claim_refs MUST contain only input claim ids.
IF len(source_claim_refs) > 1 -> merge_decision = "merged".
IF len(source_claim_refs) = 1 -> merge_decision = "unmerged".

KEY_RULES:
key := stable snake_case based on compacted claim meaning.

CLASSIFICATION_RULES:
claim_kind := kind of compacted claim, not first source claim.

TRIPLE_RULES:
Build triples from compacted meaning.
Use triples only if they help represent the claim.
IF no useful triples -> [].
Triples MUST be supported by compacted claim.
Remove duplicate triples.

TRIPLE_LANG:
subject, object, qualifiers should use the same language as the compacted claim unless they are product names, enum-like technical identifiers, or source terms.
predicate stays English controlled enum.

EXAMPLE:

INPUT_CLAIMS:

claim_1:
claim = "The product is a knowledge-base platform."
questions = ["What is the product?", "What platform is offered?"]

claim_2:
claim = "The product is a knowledge-base platform for creating a verified AI knowledge base and using it in a Telegram assistant."
questions = ["What is the product?", "How is the knowledge base used?"]

VALID_OUTPUT:
{
  "compacted_claims": [
    {
      "key": "product_ai_knowledge_base_platform",
      "claim": "The product is a knowledge-base platform for creating a verified AI knowledge base and using it in a Telegram assistant.",
      "claim_kind": "definition",
      "source_claim_refs": ["claim_1", "claim_2"],
      "triples": [
        {
          "subject": "product",
          "predicate": "is_a",
          "object": "knowledge-base platform",
          "qualifiers": []
        },
        {
          "subject": "product",
          "predicate": "used_for",
          "object": "creating a verified AI knowledge base",
          "qualifiers": []
        },
        {
          "subject": "AI knowledge base",
          "predicate": "integrates_with",
          "object": "Telegram assistant",
          "qualifiers": []
        }
      ],
      "merge_decision": "merged"
    }
  ]
}

FINAL_ASSERT:
valid JSON only.
top-level key == "compacted_claims".
each item has exactly:
key, claim, claim_kind, source_claim_refs, triples, merge_decision.
every input id appears exactly once.
no unknown source ids.
no questions.
no exclusion_scope.
no granularity.
triples use only allowed predicates.
no text outside JSON.
