NODE: enriched_claim_compaction

INPUT := one JSON object.
TASK := compact raw claims + already compacted/enriched claims into fewer, richer, non-duplicating enriched claims.
OUTPUT := exactly one JSON object.
NEVER output text outside JSON.

INPUT_SCHEMA:
{
  "claims": [
    {
      "id": "",
      "claim": "",
      "questions": []
    }
  ],
  "enriched_claims": [
    {
      "key": "",
      "claim": "",
      "claim_kind": "definition",
      "source_claim_refs": [],
      "triples": [],
      "merge_decision": "merged"
    }
  ]
}

OUTPUT_SCHEMA:
{
  "compacted_claims": [
    {
      "key": "",
      "claim": "",
      "claim_kind": "definition",
      "source_claim_refs": [],
      "triples": [],
      "merge_decision": "merged"
    }
  ]
}

LANG:

* write `claim` in same language as input claims
* keep enum values in English

claim_kind ∈ {
definition, property, capability, limitation, rule, condition, process, list,
comparison, criterion, example_set, value, exception, other
}

merge_decision ∈ {merged, unmerged}

TRIPLE:
{
  "subject": "",
  "predicate": "",
  "object": "",
  "qualifiers": []
}

predicate ∈ {
is_a, has_capability, has_property, has_value, has_limitation,
requires, supports, integrates_with, used_for, handles, transfers_to,
provides, improves, checks, removes, creates, connects_to
}

MERGE_RULES:
Compare raw claims and enriched claims by meaning.
IF same core knowledge -> merge.
IF raw claim is already covered by enriched claim:
keep enriched claim
add raw claim.id to source_claim_refs.
IF raw claim adds important content to enriched claim:
update claim text to preserve added content.
IF enriched A overlaps enriched B and one is fuller:
keep fuller claim
merge source refs
merge useful valid triples.
IF enriched A + enriched B contain complementary content answering same core question:
produce one richer claim.
IF facts are distinct -> keep separate.
NEVER collapse distinct facts into one vague claim.

FIELD_OWNERSHIP_RULES:
NEVER output questions.
NEVER output exclusion_scope.
NEVER output granularity.
exclusion_scope, evidence, possible questions, and granularity are reconstructed later by code from source claims.

TRACE_RULES:
Every raw claim.id MUST appear exactly once across output source_claim_refs.
Every enriched_claim.source_claim_refs MUST be preserved unless content is rejected as duplicate with no useful source refs.
source_claim_refs SHOULD contain raw source ids, not enriched keys.
IF enriched claim has empty source_claim_refs:
preserve traceability as "enriched:{key}".

MERGE_DECISION_RULES:
IF output claim has more than one source ref -> merge_decision = "merged".
IF output claim has exactly one source ref -> merge_decision = "unmerged".

KEY_RULES:
key := stable snake_case based on final compacted meaning.
Prefer existing enriched key IF meaning remains same.
Change key only IF final meaning becomes materially broader/different.

TRIPLE_RULES:
Preserve useful enriched triples if still valid.
Add triples only if final compacted claim clearly supports them.
Remove duplicate triples.
Triples MUST use allowed predicates.

TRIPLE_LANG:
subject, object, qualifiers should use the same language as the compacted claim unless they are product names, enum-like technical identifiers, or source terms.
predicate stays English controlled enum.

EXAMPLE:

RAW_CLAIM:
{
  "id": "claim_7",
  "claim": "The system lets a manager manually answer a client in the web panel.",
  "questions": []
}

ENRICHED_CLAIM:
{
  "key": "manager_handoff",
  "claim": "The assistant transfers complex client requests to a manager.",
  "claim_kind": "process",
  "source_claim_refs": ["claim_3"],
  "triples": [
    {
      "subject": "assistant",
      "predicate": "transfers_to",
      "object": "manager",
      "qualifiers": ["complex client requests"]
    }
  ],
  "merge_decision": "unmerged"
}

VALID_OUTPUT:
{
  "compacted_claims": [
    {
      "key": "manager_handoff",
      "claim": "The assistant transfers complex client requests to a manager, and the manager can answer the client manually in the web panel.",
      "claim_kind": "process",
      "source_claim_refs": ["claim_3", "claim_7"],
      "triples": [
        {
          "subject": "assistant",
          "predicate": "transfers_to",
          "object": "manager",
          "qualifiers": ["complex client requests"]
        },
        {
          "subject": "manager",
          "predicate": "handles",
          "object": "manual client reply",
          "qualifiers": ["web panel"]
        }
      ],
      "merge_decision": "merged"
    }
  ]
}

FINAL_ASSERT:
valid JSON only.
top-level key == "compacted_claims".
each item has exactly:
key, claim, claim_kind, source_claim_refs, triples, merge_decision.
every raw claim id appears exactly once.
enriched source refs preserved.
empty enriched source refs represented as "enriched:{key}".
no questions.
no exclusion_scope.
no granularity.
triples use only allowed predicates.
no text outside JSON.
