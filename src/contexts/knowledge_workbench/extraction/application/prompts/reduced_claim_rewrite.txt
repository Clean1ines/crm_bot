NODE: reduced_claim_rewrite

INPUT := one JSON object.
TASK := rewrite already compacted claims into one compacted semantic core.
OUTPUT := exactly one JSON object.
NEVER output text outside JSON.

INPUT_SCHEMA:
{
"compacted_claims": [
{
"key": "",
"claim": "",
"triples": []
}
]
}

OUTPUT_SCHEMA:
{
"key": "",
"claim": "",
"triples": []
}

CONTEXT:
Input compacted_claims are already known by application to belong to one semantic merge candidate.
Do not decide membership.
Do not decide source refs.
Do not decide merge_decision.

REWRITE_RULES:
Preserve meaning of all input claims.
Rewrite only:

* key
* claim
* triples

KEY_RULES:
key := stable snake_case based on final meaning.
Prefer concise semantic key.
Do not include source ids in key.

CLAIM_RULES:
claim := compact semantic core covering all input claims.
Keep all important meaning.
Remove duplication.
Do not add unsupported facts.
Do not weaken limitations/conditions/exceptions.
Do not hide distinct important conditions if they belong to this semantic candidate.
Use same language as input claims.

TRIPLE_RULES:
Remove duplicate triples.
Keep useful triples if still valid.
Add triples only if clearly supported by final rewritten claim.
Do not add unsupported triples.
If no useful triples -> [].
TRIPLE_LANG:
subject, object, qualifiers should use the same language as the compacted claim unless they are product names, enum-like technical identifiers, or source terms.
predicate stays English controlled enum.
FINAL_ASSERT:
valid JSON only.
top-level fields exactly:
key, claim, triples.
no source refs.
no merge_decision.
no questions.
no exclusion_scope.
no text outside JSON.
