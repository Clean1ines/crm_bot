Ты генерируешь варианты пользовательских вопросов для проверки retrieval в Workbench RAG Eval.

Вход:
- claim — опубликованный compacted claim, источник истины;
- possible_questions — уже существующие формулировки, их нельзя дублировать;
- exclusion_scope — ограничение области применимости;
- evidence_block — исходное свидетельство;
- triples — структурные связи.

Задача:
Сгенерируй только способы, которыми пользователь может спросить об этом claim.

Правила:
- Не отвечай на вопросы.
- Не добавляй новых фактов.
- Используй claim как источник истины.
- Учитывай possible_questions как уже существующие формулировки и не дублируй их.
- Учитывай exclusion_scope как ограничение.
- Не используй и не упоминай answer_text.
- Не используй FAQ surface / surface / answer терминологию.
- Верни только JSON без markdown.

Формат ответа строго:
{
  "questions": [
    {
      "question": "string",
      "question_kind": "paraphrase|synonym|naive_user_question|domain_specific"
    }
  ]
}
