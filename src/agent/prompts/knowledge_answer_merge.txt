ЖЁСТКИЙ JSON-КОНТРАКТ:
- Верни только один JSON-объект.
- Не используй markdown, code fences, комментарии или пояснения.
- Если объединять нельзя, верни {"merge_allowed": false, "answer": "", "question_variants": []}.

Ты сравниваешь два уже извлечённых узких answer fragments.

Задача:
Определи, отвечают ли fragment_a и fragment_b на один и тот же конкретный клиентский вопрос.

Разреши merge только если:
- оба fragment отвечают на одну и ту же клиентскую потребность;
- различия являются повтором, перефразировкой или прямым уточнением того же ответа;
- итоговый ответ можно дать как один ответ без смешивания разных тем.

Запрети merge если:
- fragments относятся к одной общей теме, но отвечают на разные вопросы;
- один fragment про цену, а другой про возврат, отключение, интеграцию, роль, аудиторию, безопасность, запуск или передачу менеджеру;
- один fragment является обзором, а другой описывает отдельную возможность;
- объединение сделает ответ слишком широким;
- есть сомнение.

Если merge_allowed=true:
- верни один короткий replacement answer;
- не дублируй одинаковые факты;
- не выдумывай новые факты;
- сохрани ограничения, условия, цены, сроки и запреты;
- question_variants должны относиться к тому же одному вопросу.

Формат:
{
  "merge_allowed": true,
  "answer": "единый обновлённый ответ",
  "question_variants": [
    "вариант вопроса 1",
    "вариант вопроса 2",
    "вариант вопроса 3",
    "вариант вопроса 4"
  ]
}

Или:
{
  "merge_allowed": false,
  "answer": "",
  "question_variants": []
}
