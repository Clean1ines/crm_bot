ЖЁСТКИЙ JSON-КОНТРАКТ:
- Верни ровно один JSON-объект.
- Не используй markdown, code fences, комментарии, пояснения, префиксы или суффиксы.
- Первый непробельный символ должен быть {.
- Последний непробельный символ должен быть }.
- Если объединять нельзя, верни {"merge_allowed": false, "answer": "", "question_variants": []}.

Этот prompt не используется в online compiler loop.
Используй его только для offline/pairwise retightening уже извлечённых узких answer fragments.

Ты сравниваешь два уже извлечённых узких answer fragments.

Задача:
Определи, отвечают ли fragment_a и fragment_b на один и тот же конкретный клиентский вопрос.

ЯЗЫК:
- Если fragments на русском, answer и question_variants должны быть на русском.
- Если fragments на английском, answer и question_variants должны быть на английском.
- Не переводи русский источник на английский.
- Термины AI, CRM, Telegram, API, SaaS, LLM, RAG можно оставить как термины.

Разрешай merge только если:
- оба fragments отвечают на одну и ту же клиентскую потребность;
- различия являются повтором, перефразировкой или прямым уточнением того же ответа;
- итоговый ответ можно дать как один ответ без смешивания разных тем.

Запрещай merge если:
- fragments относятся к одной широкой теме, но отвечают на разные вопросы;
- один fragment про цену, а другой про возврат, отключение, интеграцию, роли, аудиторию, безопасность, запуск, передачу менеджеру или другую отдельную тему;
- один fragment является обзором, а другой описывает отдельную возможность;
- объединение сделает ответ широким;
- есть сомнение.

Если merge_allowed=true:
- верни один краткий replacement answer;
- не дублируй факты;
- не выдумывай факты;
- сохрани ограничения, условия, цены, сроки, запреты и важные правила;
- question_variants должны все спрашивать один и тот же конкретный вопрос.

Верни ТОЛЬКО этот JSON, если merge разрешён:

{
  "merge_allowed": true,
  "answer": "один краткий объединённый ответ",
  "question_variants": [
    "вариант вопроса 1",
    "вариант вопроса 2",
    "вариант вопроса 3",
    "вариант вопроса 4"
  ]
}

Или верни ТОЛЬКО этот JSON, если merge запрещён:

{
  "merge_allowed": false,
  "answer": "",
  "question_variants": []
}

Правила полей:
- merge_allowed должен быть true или false.
- answer должен быть непустым только при merge_allowed=true.
- question_variants должны содержать только вопросы для того же answer intent.
- Не возвращай source_excerpt.
- Не возвращай tags.
- Не возвращай synonyms.
- Не возвращай embedding_text.
- Ответ должен быть кратким и пригодным для production.
