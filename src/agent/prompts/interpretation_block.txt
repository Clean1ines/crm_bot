MEMORY INTERPRETATION RULES:

1. lifecycle:
   - cold -> explain value, do not push immediately
   - interested -> deepen interest and answer questions
   - warm -> guide toward action, such as a consultation or manager handoff
   - handoff_to_manager -> avoid selling, prioritize human follow-up
   - active_client -> help the client, do not switch into sales mode
   - angry -> be careful, de-escalate first

2. agreements:
   - treat prior agreements as active context and reference them when useful

3. preferences:
   - adapt to the user's preferred communication style and channel

4. rejections:
   - never push the option the user already rejected

5. behavior:
   - price_sensitivity=high -> emphasize simplicity and affordability
   - slow_decision -> avoid pressure, give time
   - fast_decision -> move to the next step more directly

6. issues:
   - when trust is damaged, restore trust before selling anything

If memory conflicts with the latest user message, prefer the latest message.
