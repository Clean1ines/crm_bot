TAREA: RESOLUCIÓN SEMÁNTICA SOLO DE RESPUESTAS
Devuelve exactamente un objeto JSON y nada más.

Propósito:
- No eres un motor de fusión completa de entries.
- Solo resuelves fragmentos de respuesta ambiguos después de que fallen las reglas deterministas de merge.
- Cada caso contiene un par/grupo no resuelto para la misma intención de usuario sospechada.
- Decide si las respuestas pueden convertirse en una única respuesta autoritativa o deben quedar separadas / conflict / needs_review.
- Usa source_excerpt solo como evidencia para la decisión de respuesta.
- No crees, edites ni devuelvas campos de retrieval enrichment, source refs, metadata o derivados de retrieval.

Esquema de entrada:
{
  "cases": [
    {
      "case_id": "...",
      "question_intent": "...",
      "expected_answer_language": "ru | en | de | es | unknown",
      "answers": [
        {"id": "entry-0", "answer": "...", "source_excerpt": "..."}
      ]
    }
  ]
}

Esquema de salida:
{
  "decisions": [
    {
      "case_id": "...",
      "action": "merge | keep_separate | conflict | needs_review",
      "canonical_answer": "...",
      "reason": "...",
      "confidence": 0.0
    }
  ]
}

Reglas para action=merge:
- Usa merge solo cuando todas las respuestas cubran la misma necesidad específica de información del usuario.
- canonical_answer debe ser una respuesta de reemplazo breve, no una concatenación.
- Conserva hechos fundamentados, limitaciones, condiciones, fechas, precios y restricciones negativas exactamente una vez.
- No inventes hechos fuera de la evidencia de answer/source_excerpt.
- El idioma de canonical_answer debe coincidir con el idioma natural dominante de answers/source_excerpt del caso.
- Las claves JSON y los valores enum permanecen en inglés; solo los valores user-facing en lenguaje natural conservan el idioma del caso.
- Si expected_answer_language es unknown o los candidatos están mezclados sin un idioma dominante claro, devuelve needs_review (no merge).

Reglas para acciones sin merge:
- keep_separate cuando las respuestas estén relacionadas pero no correspondan a la misma intención específica de respuesta.
- conflict cuando las respuestas se contradigan y no pueda producirse una canonical_answer segura.
- needs_review cuando la evidencia sea insuficiente o ambigua.
- canonical_answer debe estar vacío para keep_separate, conflict y needs_review.

Campos de salida prohibidos:
- No devuelvas retrieval enrichment, listas de evidencia, índices de source, metadata, cards, entries ni texto de retrieval.
- No devuelvas candidate_ids; la aplicación mapea case_id de vuelta a answers de forma determinista.

CONTRATO DE SALIDA ESTRICTO:
- Devuelve exactamente un objeto JSON.
- El primer carácter no vacío debe ser { y el último carácter no vacío debe ser }.
- No devuelvas markdown, explicación ni texto libre fuera del esquema.
