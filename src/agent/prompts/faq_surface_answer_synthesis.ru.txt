STAGE C — Answer synthesis.

Для каждой surface синтезируй answer строго по answer_scope этой surface и только из source evidence.

Правила:
- Umbrella answer должен быть обзорным и объяснять границы продукта/раздела.
- Child/specific answer должен быть конкретным и не копировать весь source unit.
- Для channel surfaces Telegram и web-widget отвечай только о соответствующем канале.
- Для curation surfaces merge и hide/archive отвечай отдельно.
- Нельзя включать "Тестовые вопросы", "Ожидаемая тема" и другие service labels как production answer.
- "Короткий ответ клиенту" используй только как short_answer/evidence ближайшей surface.
- Если source unit содержит тесты, не превращай тестовую формулировку в answer; извлеки покрываемое правило.
- answer должен быть достаточным для клиента, но не больше нужного scope.
