7e9492c 2026-03-20 clean1ines

  refactor(agent): replace router with intent-policy-response pipeline

  This major refactoring replaces the monolithic router_llm node with a
three-stage pipeline for more modular and controllable AI behavior:

1. Intent Extractor (lightweight LLM)
   - Detects user intent (pricing, support, sales, feedback)
   - Extracts call-to-action type (request_demo, call_manager, etc.)
   - Identifies mentioned features with interest scores
   - Uses llama-3.1-8b-instant for speed and cost efficiency

2. Policy Engine (pure Python rule engine)
   - Applies business logic based on current lifecycle stage and intent
   - Implements transition table for lifecycle (cold → interested → warm → hot)
   - Decides next action: LLM_GENERATE, ESCALATE_TO_HUMAN, CALL_TOOL
   - Overrides decisions based on high feature interest scores

3. Response Generator (powerful LLM)
   - Uses main Groq model (configured via settings)
   - Crafts final answer using all context (KB, memory, features, CTA)
   - Produces natural, sales-oriented responses

Additional improvements:
- Load_state now fetches analytics fields (intent, lifecycle, cta, decision)
  from threads table and user memory, merging both sources
- Persist node now updates analytics fields in threads table
- Rules node simplified to only anger detection (cheap routing)
- Added IntentOutput Pydantic schema for validation
- All new nodes use log_node_execution for observability

Graph flow:
load_state → rules → kb_search → intent_extractor → policy_engine
         → (if LLM_GENERATE) response_generator → responder → persist
         → (if ESCALATE) escalate → responder → persist
         → (if CALL_TOOL) tool_executor → response_generator → responder → persist

This change improves:
- Separation of concerns (intent detection, policy, response generation)
- Cost efficiency (lightweight model for intent, heavy only for final answer)
- Analytics readiness (intent, lifecycle, CTA, decision stored per thread)
- Maintainability (business rules in Python, not LLM prompt)
